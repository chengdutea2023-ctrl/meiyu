# 第一阶段 MVP 范围

## 目标

第一阶段先实现统一登录和业务应用接入跑通，让多个独立业务应用可以共用一套智美教育新生态业务底座账号系统。

业务应用有自己的服务器和入口，但不直接读取底座数据库。业务应用通过标准 API 和 SDK 获取用户、组织、班级等上下文。

## 范围

### 统一账号

- 用户名、邮箱、密码登录
- 密码加密存储
- 用户启用、禁用
- 当前用户信息 API

### 统一登录

- 底座提供统一登录页
- 业务应用跳转到底座登录页
- 登录成功后通过一次性 code 回跳业务应用
- 业务应用服务端使用 `code + appSecret` 换取 token
- 业务应用使用 token 调用底座 API

### 业务应用接入

- 管理员登记业务应用
- 每个业务应用拥有 `appId` 和 `appSecret`
- 配置业务应用名称、入口地址、回调地址
- 校验回调地址，避免任意跳转
- 提供业务应用接入文档

### 基础教育模型

- 机构 / 学校
- 班级
- 用户
- 角色
- 用户所属机构
- 用户所属班级

### 管理后台

- 管理员登录
- 用户管理
- 机构管理
- 班级管理
- 业务应用管理

## 非目标

第一阶段暂不实现：

- 微信支付
- 支付宝支付
- 订单中心
- 权益中心
- 复杂插件市场
- 微信登录
- 手机号登录
- 第三方 OAuth 登录
- 复杂数据看板

## 数据库选择

第一阶段主数据库使用 PostgreSQL。

原因：

- 适合用户、组织、班级、角色、业务应用等强关系数据
- 支持事务、唯一约束、外键和复杂查询
- 支持 JSON 字段，方便保留部分扩展配置
- 适合后续扩展订单、权益、审计日志等严肃业务数据

Redis 只用于缓存和短期状态，例如：

- 一次性登录 code
- refresh token 状态
- 登录限流
- 验证码
- 临时会话状态
